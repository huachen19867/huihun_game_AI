# AI 游戏复刻提示词文档：回煞 (Hui Sha)

如果你想让 AI 帮你复刻这个游戏，你可以直接使用下面的提示词。我已经将核心架构、玩法机制和技术细节拆解成了结构化的 Prompt。

---

## 角色设定
你是一个精通 **Phaser 3** 引擎和 **Web 游戏开发** 的资深全栈工程师。你需要帮我开发一款名为《回煞》的 **2D 俯视角中式恐怖解谜游戏**。

## 1. 技术栈要求
- **核心引擎**: Phaser 3 (使用 Arcade Physics 物理系统)
- **语言**: JavaScript (ES6+ 模块化开发)
- **平台**: Web 浏览器 (需完美适配 PC 和 移动端)
- **渲染**: Canvas/WebGL (需包含 Light2D 动态光照效果)

## 2. 游戏核心设计

### A. 氛围与题材
- **背景**: 80年代中式农村老宅，头七回魂夜。
- **视觉风格**: 像素风/低保真，黑暗压抑，使用动态光影（手电筒光锥）来限制视野。
- **核心玩法**: 探索房间 -> 收集供品/线索 -> 解开机关 -> 躲避怪物 -> 达成结局。

### B. 地图系统 (`MapManager`)
- 不要依赖外部 Tiled 编辑器，请设计一个基于 **二维数组** 的地图生成系统。
- **地图数据结构**: 0代表地面，1代表墙壁。
- **多场景切换**: 支持从“正厅”切换到“走廊”、“卧室”、“后院”等，切换时要有淡入淡出效果。
- **对象配置**: 在地图数据中配置 NPC、交互物（棺材、供桌、柜子）的坐标和属性。

### C. 玩家控制 (`Player`)
- **PC端**: WASD 移动，空格/E 键交互。
- **移动端**:
  - 左侧屏幕：虚拟摇杆 (Joystick) 控制移动。
  - 右侧屏幕：交互按钮 (Action Button)。
  - 界面需自适应，防止 UI 遮挡。
- **手电筒**: 玩家自带光源，随移动方向旋转，周围环境黑暗。

### D. 交互与叙事 (`InteractionManager`)
- **距离检测**: 玩家靠近物体出现提示。
- **对话系统**: 打字机效果显示文本，支持多段对话回调。
- **背包系统**: 简单的格子背包，点击物品可查看详情。
- **关键道具**: 倒头饭、香、纸钱、火柴（用于供奉）；钥匙（开启房门）。

### E. 恐怖机制 (重点)
1. **Sanity (理智值)**:
   - 在黑暗中或靠近怪物时理智下降。
   - 理智过低触发：屏幕抖动、甚至出现幻听（随机音效）。
2. **追逐战 (The Chaser)**:
   - 触发条件：捡起“血红钥匙”后生成黑色鬼影。
   - AI逻辑：简单的寻路算法，持续追踪玩家。
3. **躲藏系统**:
   - 玩家可以躲进柜子或床底。
   - **麦克风互动 (创新点)**: 躲藏时调用浏览器麦克风权限 (`navigator.mediaDevices`)。如果玩家现实中发出声音超过阈值，怪物会直接发现并处决玩家。

## 3. 逐步开发计划 (Step-by-Step)

请按照以下步骤生成代码：

**第一步：基础框架**
- 创建 `index.html`，包含 Phaser 初始化和自适应 CSS。
- 实现 `GameScene`，加载基础素材（用色块代替占位）。
- 实现基于数组的地图渲染和玩家移动。

**第二步：交互系统**
- 实现 `InteractionManager`。
- 添加 UI 层（对话框、背包栏）。
- 实现简单的钥匙开门逻辑。

**第三步：恐怖氛围**
- 加入 `Light2D` 管道，实现手电筒光照。
- 添加 `SoundManager`，支持空间音效（距离越近声音越大）。
- 实现供桌解谜：集齐4样供品点燃蜡烛。

**第四步：怪物与躲藏**
- 实现怪物寻路 AI。
- 实现躲藏逻辑（玩家隐身、失去控制权）。
- **关键**: 编写麦克风检测逻辑，将音量映射到游戏内的“暴露度”。

**第五步：结局与优化**
- 根据背包物品判定结局（普通结局 vs 真结局）。
- 移动端触摸优化（防误触、摇杆手感）。

## 4. 关键场景描述 (Visual References)

为了确保游戏氛围到位，请参考以下关键场景的视觉描述进行开发：

**场景一：灵堂供桌 (The Altar)**
- **描述**: 昏暗的正厅中央，摆放着一张红木供桌。桌上放着黑白遗像（被黑布遮住），两旁是未点燃的蜡烛。
- **光影**: 只有玩家手电筒的冷光束打在供桌上，四周是深邃的黑暗。
- **氛围**: 死寂，只有偶尔传来的风声。

**场景二：血红追逐 (The Chase)**
- **描述**: 玩家捡起血红钥匙的瞬间，周围环境泛起红光。一个黑色的、模糊的人形鬼影从角落生成，向玩家高速移动。
- **特效**: 屏幕边缘有红色的晕影 (Vignette) 闪烁，镜头剧烈晃动，伴随着刺耳的噪音。

**场景三：柜中窥视 (Hiding)**
- **描述**: 玩家躲进衣柜，视野被限制在一条狭窄的缝隙中。
- **交互**: 屏幕中央出现麦克风图标，随着现实声音的大小而波动。如果声音太大，图标变红。
- **视角**: 透过缝隙可以看到怪物在外面徘徊的影子。

**场景四：枯井惊魂 (The Well)**
- **描述**: 后院的枯井旁，杂草丛生。井口贴着黄色的符纸。
- **事件**: 当玩家靠近时，一只苍白的手突然抓在井沿上。
